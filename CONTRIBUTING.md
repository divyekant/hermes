# Contributing to Hermes

Thanks for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feat/my-feature`)
3. Make your changes
4. Run tests before committing
5. Commit with conventional style (`feat:`, `fix:`, `docs:`, etc.)
6. Open a pull request

## Development

Stack and setup instructions will be added once the project architecture is finalized.

## Code Style

- Concise, minimal comments
- Design before code — brainstorm/shape the approach first
- TDD — write tests before implementation

## Reporting Issues

Use the GitHub issue templates for bug reports and feature requests.
