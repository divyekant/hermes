# Hermes

## The Problem

Every time your engineering team ships a feature, a translation tax kicks in. Customer support discovers new features from customers asking "what does this do?" Marketing writes about it from Slack threads and secondhand information. Documentation falls behind within days. The knowledge exists in the code — but it is locked in engineering-speak, and every other team has to manually translate it.

## The Solution

Hermes reads your codebase and generates documentation for three audiences simultaneously: detailed handoffs for support teams, user-facing guides for developers, and benefit-focused briefs for marketing. One source, three tailored outputs. No manual translation, no waiting for engineering to "write it up."

## Key Benefits

- **Close the knowledge gap instantly:** CS gets feature handoffs with edge cases, troubleshooting steps, and anticipated customer questions — generated the moment a feature ships
- **Marketing materials at release speed:** Feature briefs, sales one-pagers, and release announcements are ready within minutes, not days of reverse-engineering
- **Documentation that stays current:** Incremental update mode regenerates only what changed, so docs evolve with your code instead of falling behind
- **Zero setup required:** Works on any project from the first run. No configuration, no schemas, no upfront investment.

## How It Works

1. **Analyze** — Hermes reads your code, docs, and git history to understand what was built
2. **Generate** — It produces three tailored document sets: internal (CS), external (developers), and marketing (sales/PMM)
3. **Update** — On subsequent runs, it detects what changed and regenerates only the affected docs

## Who It's For

- **Engineering leads** who are tired of answering the same questions from CS and marketing after every release
- **CS/Support managers** who want their team to answer customer questions without escalating to engineering
- **Product marketers** who need accurate feature descriptions without scheduling meetings with developers
- **Developer relations** teams managing open-source projects that need quality user docs without a dedicated tech writing team

## Get Started

Add Hermes to your project and say "Generate docs for this project" in a Claude Code session. Documentation for all three audiences in minutes.
